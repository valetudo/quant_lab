## 4. Quality Stocks refinement — FINAL VERDICT

🟢 **V5 (`quality_stocks_v5_combined`) is the recommended variant.**

| Variant | Median OOS Sharpe | p25 | p75 | Verdict |
|---|---:|---:|---:|---|
| baseline | **-0.208** | -0.269 | -0.146 | 🔴 OVERFIT |
| V2 — faster trend (20/100) | +0.911 | -0.172 | +1.401 | 🟡 MARGINAL |
| V3 — shorter momentum (60/5) | +0.014 | -0.441 | +0.227 | 🔴 OVERFIT |
| V4 — no bond fallback | +0.636 | +0.225 | +1.430 | 🟢 ROBUST |
| **V5 — V2+V3+V4 combined** | **+0.745** | **+0.480** | +0.779 | **🟢 ROBUST** |

V5 picked over V4 because **p25 is +0.480** (highest of all variants) and **the p75−p25 spread is narrowest (+0.30)** — every fold is positive or near-zero. V4 has fold 3 with NaN Sharpe (zero trades), which is operationally awkward.

The dominant lever was the **bond fallback**: replacing IEF parking with cash in bear regime alone cleared ROBUST (V4). The faster trend (V2) helped fold 4 + 5 substantially. The shorter momentum (V3) was the weakest individual change.

**Caveat**: V5's promotion is **provisional**. The variants were chosen with prior knowledge of the baseline's per-fold weaknesses, so confirmation bias is non-zero. Full reasoning + honest list of unknowns lives in [QUALITY_STOCKS_REFINEMENT_REPORT.md](QUALITY_STOCKS_REFINEMENT_REPORT.md).

## 6. Autonomous decisions

1. **Kept the old `MasterAllocator` / `aggregator` modules importable** via thin re-export stubs that emit a `DeprecationWarning`. Physical files moved to `portfolio/_legacy/`. This avoided breaking the 7 existing tests in `test_portfolio_aggregator.py` while still satisfying the spec's "sposta in `_legacy/`" instruction.
2. **Phase 3 modified `regime.is_market_uptrend` callers, not the function itself.** Wired `trend_short_ma` / `trend_long_ma` from the YAML config through a small `_trend_kwargs()` helper in the strategy. The function signature already accepted them — they just weren't being passed. Without this, V2 and V5 would have silently been identical to baseline.
3. **Auto-recommendation in the UI** picks the variant with the **highest p25** within the ROBUST tier (or MARGINAL if no ROBUST), tie-broken by median. This favours consistency over peak performance — matches the user's stated preference for not "saving a strategy" by tweaking.
4. **The baseline `walk_forward_verdict.json` at the legacy location is preserved.** `run_quality_walk_forward.py --variant baseline` writes both to `walkforward_baseline/walk_forward_verdict.json` (new layout) AND to the legacy single-file path that the page-7 banner used to read. So a fresh-clone install that only runs the baseline still gets a coherent banner.
5. **Streamlit autorefresh is required for Live mode** (Phase 2.5) — Phase 3 added the `pip install streamlit-autorefresh` line to the README's quick-start, but does NOT change `pyproject.toml` (the dependency is still soft-imported with a fallback message).
6. **Bond Ladder cash-flow projection assumes annual coupon on the maturity anniversary**. Marked clearly "MOCK DATA assumption" in the UI caption + ladder docstring. Real coupon schedules are a Phase 4 data-feed task — flagged in the TODO.
7. **The opportunistic sleeve sits as cash** (10% × total capital) until a strategy is wired in. This is the explicit user policy ("cash in EUR, not migrated to bonds"), enforced by `PortfolioState._opportunistic_sleeve_value`.
8. **Page-1 Equity tab now picks the best-of-N variant verdict**, not the legacy verdict file alone. Otherwise it would have shown OVERFIT (baseline) while page-7 showed the V5 ROBUST banner — inconsistent UX.

## 7. Technical trade-offs

1. **Walk-forward verdict thresholds inherited unchanged**: `>= 0.40` (ROBUST), `>= 0.20` (MARGINAL). These are conservative heuristics, not p-values. With 5 folds the statistical power is limited; a wider validation window (10+ folds, or a 2014-2019 backtest) is the proper next step (Phase 4).
2. **`PortfolioState` re-reads parquet on every UI rerun** (capped by a 30 s `st.cache_data` TTL). Fine for the current ladder size (<100 positions); could move to a singleton or to DuckDB if/when the ladder grows to hundreds of bonds.
3. **Bond Ladder is single-currency UX** (EUR by default). The `currency` selector is wired in candidate filtering but `current_market_value_eur` does not convert from non-EUR notional. Acceptable for the Italian/EU bond focus; would need FX integration if we add US/UK bonds at scale.
4. **No automated regression test for "Live mode produces standard outputs"** — the streaming flow is verified by manual smoke + AppTest probing the page. A proper `streamlit.testing.v1` cancellation test would need fixtures for the thread + fake panel; deferred.
5. **Walk-forward fold 1 Sharpe drift remains**: Phase 2 reported +1.87 for baseline fold 1, Phase 2-fix +1.76, Phase 3 baseline +1.76 again. Other folds match to 2 decimals. Likely cache state — flagged in QUALITY_STOCKS_REFINEMENT_REPORT.md as a LOW-priority Phase 4 investigation.
6. **Deprecation warnings on every aggregator/master_allocator import**. Existing tests now emit `DeprecationWarning: portfolio.aggregator is deprecated`. They still pass; the warning is the desired UX. New code should use `portfolio.static_allocator`.
